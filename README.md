```
[기획] → [스토리보드] → [이미지 생성] → [영상 생성] → [오디오] → [후반편집] → [최종 검수]
  ↓           ↓               ↓               ↓            ↓           ↓
브랜드    씬별 구성       GPT Image 2      Sora 2      기삽입     자막·CTA    배포 준비
정의      타임코드        키프레임 3장     10초 영상   오디오     하드코딩
```

### 설계 근거
1. **기획 선행:** 메시지·타겟·톤 확정 없이 생성 시 방향성 분산 위험
2. **스토리보드 → 이미지 순서:** 씬 구조 확정 후 키프레임 생성으로 일관성 확보
3. **이미지 → 영상 순서:** 키프레임을 레퍼런스로 Sora 2에 제공, 스타일 연속성 유지
4. **오디오 후처리:** 영상 완성 후 오디오 싱크 맞춤이 역순보다 효율적
5. **자막 분리:** AI 도구의 한국어 자막 정확도 한계 → 후반 편집으로 품질 보장

## 평가기준 12 · Text-to-Image와 Image-to-Video 선택 기준

### 개념 정의
| 구분 | Text-to-Image (T2I) | Image-to-Video (I2V) |
|---|---|---|
| **정의** | 텍스트 프롬프트만으로 정지 이미지를 생성 | 정지 이미지를 입력값으로 영상(모션)을 생성 |
| **입력값** | 텍스트 프롬프트 | 이미지 + 텍스트 프롬프트 |
| **출력값** | 정지 이미지 (.png / .jpg) | 영상 클립 (.mp4) |
| **대표 도구** | GPT Image 2, Midjourney, DALL·E 3 | Sora 2, Runway Gen-3, Kling |

---

### 장단점 비교
| 항목 | T2I | I2V |
|---|---|---|
| **장점** | 빠른 생성 / 크레딧 소모 적음 / 세밀한 구도·색감 제어 가능 | 키프레임 기반으로 스타일 일관성 유지 / 자연스러운 모션 구현 |
| **단점** | 정지 이미지만 생성 → 영상화 별도 필요 | 크레딧 소모 큼 / 입력 이미지 품질에 결과 의존 |
| **적합한 상황** | 키프레임·레퍼런스 이미지 확보 단계 | 확정된 키프레임을 영상으로 전환하는 단계 |

---

### 본 프로젝트 선택 근거

#### 1단계 — T2I 선택 (GPT Image 2)
> **목적:** 씬별 키프레임 확보 및 시각 스타일 확정

| 선택 이유 | 상세 |
|---|---|
| **스타일 사전 확정** | 영상 생성 전 색감·구도·조명을 이미지 단계에서 먼저 검증하여 Sora 2 생성 실패 리스크 최소화 |
| **크레딧 효율** | 이미지 생성은 영상 생성 대비 크레딧 소모가 적어 반복 수정에 유리 |
| **프롬프트 정교화** | T2I 결과물을 보며 프롬프트를 수정·확정한 뒤 I2V에 동일 지시어 재활용 가능 |

#### 2단계 — I2V 선택 (Sora 2)
> **목적:** 확정된 키프레임을 기반으로 10초 시네마틱 영상 생성

| 선택 이유 | 상세 |
|---|---|
| **스타일 연속성** | T2I로 생성한 키프레임을 레퍼런스로 제공하여 씬 간 색감·분위기 일관성 확보 |
| **모션 품질** | 텍스트 단독 입력 대비 이미지 기반 입력 시 카메라 무브·피사체 움직임 제어 정확도 향상 |
| **10초 단일 생성** | 씬 분리 생성 후 편집 방식 대비 색보정 불일치 위험 제거 |

---

### 선택 흐름 요약
```
[T2I: GPT Image 2]          [I2V: Sora 2]
텍스트 프롬프트               키프레임 이미지 + 프롬프트
      ↓                              ↓
씬별 키프레임 3장  ──────────▶  10초 시네마틱 영상
(스타일 확정 단계)              (모션 구현 단계)
